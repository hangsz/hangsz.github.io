用Scrapy shell学习XPath

发表： 2017-04-11 | 更新： 2017-04-12 | 专题： python | | 阅读次数：

版权声明：文章为博主原创，转载请注明出处。

XPath语法教程很多，配套练习教程太少。

准备

  1. 进入命令行环境，输入 scrapy shell http://scrapy.org/ –nolog
  2. 打开上述网页以用于对比，输入 view(response)

练习1

  • sel.xpath('//title').extract() # 取出以title标签开始/结束的内容。
  • sel.xpath('//title/text()').extract() #取出标签内容，text()可认为是下一级标签。
  • sel.xpath('/title') #[]，因为/开头表示从根目录开始，必须严格按照路径来匹配。

练习2

  • title = sel.xpath('/html/head/title') # 取出带标签的内容，但是类型为selector。
  • title.xpath('/text()').extract() #[]，即使指定selector， /开头也表示从根目录开始。
  • title.xpath('./text()').extract() # “.”开头表示从当前标签级别开始。
  • title.xpath('./title').extract() #[]，路径中的/表示到了下一级标签或属性。
  • title.xpath('//title').extract() # 得到完全相同的title。
  • title.xpath('title').extract() #[]，因为title没有子节点？
  • sel.xpath('html') #[]，不是应该获得所有子节点么？

练习3

  • sel.xpath("//link") # 获得 selector的列表。
  • sel.xpath("//link").extract() # 获得 selector的文本形式列表。
  • sel.xpath("//link").extract()[0] # 从列表中取出第一个。
  • sel.xpath("//link/text()") #[]，因为link没有text。

练习4

  • sel.xpath("//@ref") #[]，看来不可从任何位置匹配属性。
  • link = sel.xpath("//link/@rel") #获得属性的selector的列表。
  • sel.xpath("//link/@rel").extract() # 获得属性 selector的文本形式列表。

练习5

  • link = sel.xpath("//link")
  • len(link.xpath('./@rel')) # 20
  • len(link.xpath('//@rel')) # 400, 对于list来说，//会对每个list元素都执行一遍。
