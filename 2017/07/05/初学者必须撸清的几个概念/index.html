初学者必须撸清的几个概念

发表： 2017-07-05 | 更新： 2017-07-05 | 专题： Big Data | | 阅读次数：

版权声明：文章为博主原创，转载请注明出处。

在不涉及创新的学习中，我认为学不懂、不会用的原因只有两种：

  • 概念的定义没有明确；
  • 定义这个概念的动机没有明确。

能够精确地书面化、而不是模糊的口语化说出一个概念，并对概念中涉及的其他概念能够递归式的做到，这样可以认为是明确了概念的定义。
能够做到知道一个概念是为了解决什么问题而被提出的，它解决了这个问题没有，它带来了什么新问题，就算明白了定义概念的动机。

在ML学习中，我觉得有几个概念是很重要的，必须明确。

机器学习三要素

这个概念在李航的书里明确提出，其他书如西瓜书和PRML都没有这么细的划分。我认为这样的划分更明确，所以这里就采用了，但是具体内容又不和李航的书完全相同。
下面我们只谈监督学习，只考虑参数模型。

1.模型

按照PRML中，模型有三类：

  1. $y=f_{\theta } (x)$ ，不涉及概率的数学函数
  2. $p_{\theta}(y|x)$ ，判别模型
  3. $p_{\theta}(x,y)$ ，生成模型，一般转化为学习这两个模型：$p(x|y),p(y)$。

参数模型指在学习之前，必须给出模型的具体表达式，学习的目的就是给出其中的参数。这三种模型学习难度由低到高（表达式参数更多、需要的数据更多），相应的能够表达的信息也是由低到高，模型3可以推出2继续推出1，学习哪种类型模型需要结合实际需求来。

2. 策略

策略也即获得参数的一种套路，大多数情况下，都是将其转化为最优化问题（有约束或无约束）。

  1. 对于$y=f_{\theta } (x)$，一般转化最小平方损失问题(当然也可以用其他损失函数)，比如线性回归的最小二乘法。
  2. 对于$p_{\theta}(y|x)$，一般就是极大对数条件似然函数，比如logit回归。
  3. 对于$p_{\theta}(x,y)$ ，一般就是极大对数联合似然函数（本质上还是极大对数条件似然函数），比如朴素贝叶斯。

3. 算法

算法没什么好说的，最优化问题的一些具体计算方法。

模型选择

1. 狭义的模型选择

狭义的模型选择指的是对于同一个模型，但参数取不同的个数或者数值。

  • 模型参数个数不同。比如就是简单的线性回归，我可以取三个系数， $\omega ^{T}=(\omega ^{0},\omega ^{1},\omega ^{2} )$ ，也可以搞四个系数 $\omega ^{T}=(\omega ^{0},\omega ^{1},\omega ^{2} ,\omega ^{3} )$ ，哪个表现更好，就需要我们去选择；
  • 含有超参数，超参数指需要提前给定的参数。比如还是线性回归，我不按照第一种取不同个数去得到不同模型，现在规定就取100个系数，但是我加一个模型复杂度的罚项，如$ \lambda \omega^ {T} \omega$ ，不同的$\lambda$ ，会导致100个系数中一部分趋于0，这样就获得了跟1类似的效果。

2. 广义的模型选择

模型本身就是不同的，比如二分类问题，可以用logit回归，也可以用决策树，也可以用SVM，它们的理念出发点本身就有很大的差别。

3. 实验选择

模型如何选择，最简单的方法就是实验。在进行实验之前，会引入一个新的概念—验证集，只有在需要进行模型选择的时候才会划分验证集。
实验逻辑是这样的：将训练集划分出一部分作为验证集，选出验证集上表现好的模型，然后在全体训练集上重新训练这个模型，最后在测试集上测试其泛化性能。

不过这里有个问题，就是如果我有用的数据集太小或者说我要搞出一部分数据作为验证，那么有用的训练集就变小了，怎么办？三种处理办法被提了出来：
1. 留出法 2. 交叉验证 3.自助法

有了方法，到底哪个模型是好的，还需要给定评价指标，常用的评价指标有如下三种：
1. 错误率，精度 2. 查全率、查准率，P-R图 3. 真正例率、假正例率，ROC曲线

三种评估方法和评价指标都要记清楚，真正用起来。

4. 理论分析

泛化能力，就是指这个模型对于未知数据的预测能力。实验方法能够告诉我们模型的效果，如果能从理论上解释岂不更好？多层神经网络模型一大诟病就是其可解释性不好。

学习这块内容需要牢记：某一学习算法，不同的训练集，同一点。

  • 偏差：学习模型对于真实模型的逼近能力
  • 方差：学习模型本身有没有很大的波动
  • 噪声：永远不可能与真实的模型完全一致

学习理论

上面第4点其实就属于学习理论的范畴了。

为什么在训练集上得到的模型可以用于预测？仅仅符合直觉是不够的，还需要理论上证明之。学习理论致力于用数学语言解决以下三个问题：

  1. 如何形式化描述偏差和方差的妥协？(现在过欠拟合问题都是通过试验一点点调整的)
  2. 如何将训练误差与泛化误差联系起来？(只知道训练误差，但其实只在意泛化误差)
  3. 有没有某种条件可以确保某个学习算法一定表现良好？(现在一个问题到底用哪个算法好，总体上都是靠试验来确定的)

1. PAC学习

也即从概率角度来解释一个学习模型对真实模型的逼近程度。比如，对于问题2，不是给出一个训练误差与泛化误差差值的范围，而是给出它们差值属于某一范围的概率。

2. VC维

如果模型参数连续，那么基本的学习理论就没法使用，VC维就是解决这个问题的。假设给定一组二分类数据集，大小为$d$，那么单看$y$一共有$2^{d}$ 种组合。对于每一种组合，x随便取，只要模型空间存在一个模型能够标记它，那么就说这个模型空间可以打散-shatter这个数据集。而模型空间能够shatter的最大的那个d就是模型空间的VC维。

3. 学习理论的局限性

从Ng的课程知，能够转化为ERM经验风险最小化问题的学习算法，它的泛化误差都是可以理论分析的，但对于non-ERM的学习算法，目前还无能为力，这还是前沿问题。
